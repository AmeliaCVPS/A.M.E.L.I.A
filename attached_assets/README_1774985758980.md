# A.M.E.L.I.A.
### Atendimento Médico Eficiente e Lenitivo com Inteligência Artificial

> *"Tecnologia que Acolhe. Cuidado que Transforma."*

---

## ✨ Nossa Missão

O A.M.E.L.I.A. nasceu de uma convicção simples: **a tecnologia deve ser um instrumento de cuidado, não de distanciamento**.

Em um mundo onde o acesso à saúde ainda é desigual, acreditamos que a inteligência artificial pode ser uma ponte — não uma barreira — entre pacientes e o atendimento que precisam. Nossa plataforma une a eficiência dos algoritmos ao calor do cuidado humano, criando um espaço digital onde cada pessoa se sente vista, ouvida e acolhida.

O robô AMELIA — nossa mascote com braços abertos e sorriso acolhedor — simboliza exatamente isso: a fusão entre a precisão da tecnologia e a compaixão do cuidado médico.

---

## 🤖 A Filosofia do Design

Nossa interface foi construída sobre um princípio que chamamos de **design lenitivo**: cada elemento visual, cada cor, cada transição foi escolhida para reduzir a ansiedade e aumentar a confiança do usuário.

- **Azul profissional com tons acolhedores** — transmite confiança sem frieza
- **Tipografia clara e legível** — acessível para todas as idades
- **Feedback visual imediato** — o usuário sempre sabe o que está acontecendo
- **Linguagem humana** — nunca técnica ou intimidadora

O robô AMELIA aparece em todo o site como um símbolo de presença e cuidado — no header, no favicon, nas telas de login, nos painéis internos. Ele lembra que, por trás de cada dado de saúde, há uma pessoa que merece atenção.

---

## 🩺 Para Quem é o A.M.E.L.I.A.?

### Para Pacientes
Registre seus sintomas com detalhes, acompanhe seu histórico de saúde e receba orientações sobre a urgência do atendimento — tudo de forma simples, segura e compassiva.

### Para Profissionais de Saúde
Acesse um painel completo para monitorar pacientes, gerenciar casos por nível de urgência e priorizar o atendimento com base em dados reais e atualizados.

## 🚀 Funcionalidades

### 🔐 Autenticação Segura
- Login com **CPF** (11 dígitos) ou **Cartão SUS** (15 dígitos)
- Cadastro de novos usuários com validação completa
- Senhas criptografadas com **bcrypt** (10 rounds)
- Sessões protegidas com **JWT** (validade de 7 dias)
- Cookies HTTP-only para máxima segurança

### 👤 Dashboard do Paciente
- Visualização de estatísticas pessoais de saúde
- Histórico completo de sintomas registrados
- Registro de novos sintomas com dados vitais
- Classificação de risco visual (verde/amarelo/vermelho)
- Acompanhamento do status de cada registro

### 👨‍⚕️ Dashboard Administrativo
- Estatísticas gerais do sistema em tempo real
- Listagem e gestão de todos os pacientes
- Visualização e atualização de sintomas registrados
- Distribuição visual dos níveis de risco
- Alertas automáticos para casos críticos pendentes

### 🎯 Sistema de Triagem por Cores
| Cor | Nível | Orientação |
|-----|-------|------------|
| 🟢 Verde | Baixo Risco | Sintomas leves, monitoramento recomendado |
| 🟡 Amarelo | Médio Risco | Requer atenção médica em breve |
| 🔴 Vermelho | Alto Risco | **PROCURE ATENDIMENTO IMEDIATO** |

## ⚡ Início Rápido

### 1. Instale as dependências

```bash
pip install -r requirements.txt
```

### 2. Configure as variáveis de ambiente

```bash
cp .env.example .env
# Edite .env com suas chaves secretas
```

### 3. Popule o banco de dados

```bash
python seed.py
```

### 4. Inicie o servidor

```bash
python app.py
```

O sistema estará disponível em: **http://localhost:5000**

## 🔌 API REST

### Autenticação
```
GET  /api/auth/me        — Usuário atual (autenticado)
```

### Usuários
```
POST /api/users          — Criar usuário
GET  /api/users          — Listar usuários (apenas admin)
```

### Sintomas
```
POST  /api/symptoms               — Criar sintoma
GET   /api/symptoms               — Listar sintomas
GET   /api/symptoms?user_id=X     — Sintomas por usuário
PATCH /api/symptoms/:id           — Atualizar status
```

## 🔒 Segurança

- ✅ Senhas **nunca** armazenadas em texto plano (bcrypt 10 rounds)
- ✅ Tokens JWT com expiração automática de 7 dias
- ✅ Cookies HTTP-only (não acessíveis via JavaScript)
- ✅ Validação e sanitização de todos os inputs
- ✅ Proteção de rotas por papel (admin/paciente)
- ✅ Proteção contra SQL Injection via ORM
- ✅ Verificação de CPF (11 dígitos) e SUS (15 dígitos)
